## 设计决策记录

### 2026-02-24 关键决策
1. **数据库设计**：采用13个核心表结构，不增加`articles`表
2. **Agent记忆系统**：采用代码+Markdown混合模式
3. **产出物管理**：路径记录在Agent记忆文件中，不在数据库
4. **文件命名**：统一英文蛇形命名，避免中文文件名问题
5. **目录结构**：标准化Agent目录模板

### 设计原则
1. **约定大于配置**：统一规则，减少临时决策
2. **松耦合设计**：数据库记录内容，记忆文件记录路径
3. **可视化友好**：所有配置可展示、可编辑
4. **可维护性**：清晰的文档系统，易于扩展
